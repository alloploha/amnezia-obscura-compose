# Чёрный список Obscura

`obscura-blacklist` — это дополнительный модуль для Linux-сервера с Docker.
Он блокирует нежелательные исходящие соединения контейнеров по спискам доменов и ASN.

Этот модуль работает на хосте.
Для обычного использования достаточно трёх верхнеуровневых скриптов из каталога `scripts/`.

## Что понадобится

- Linux-сервер
- Docker
- systemd
- `python3`
- root-доступ

Сетевой backend модуль выбирает сам.
Пользователю не нужно заранее разбираться, используется ли на хосте `iptables` или `nftables`.

## Установка

1. Склонируйте репозиторий:

```bash
git clone https://github.com/alloploha/amnezia-obscura-compose.git
cd amnezia-obscura-compose
```

2. Запустите установку:

```bash
sudo sh scripts/install-blacklist.sh
```

Скрипт:
- проверит наличие `python3`, `docker` и `systemctl`
- убедится, что systemd является активной init-системой
- проверит доступность Docker daemon
- установит системный launcher, конфиг, source-файлы и systemd units
- выполнит проверку
- сразу применит чёрные списки

После установки модуль использует системные пути:
- конфиг: `/etc/obscura-blacklist/blacklist.conf`
- source-файлы: `/etc/obscura-blacklist/sources/`

## Как обновлять списки

После установки редактируйте файлы именно в:

```text
/etc/obscura-blacklist/sources/
```

Например, там находятся файлы вида:
- `domains-*.txt`
- `asns-*.txt`

## Текущие списки по умолчанию

Сейчас в репозитории уже есть source-файлы чёрного списка для:
- Vkontakte
- Max Messenger
- Ростелеком
- диапазонов ASN, связанных с `gov.ru`, включая инфраструктуру, связанную с Роскомнадзором
- Сбербанк
- `mail.ru`
- Одноклассники
- Госуслуги

После изменения списков примените обновление:

```bash
sudo sh scripts/refresh-blacklist.sh
```

Этот скрипт перечитает установленный конфиг и обновит активные правила.
Он использует:
- `/etc/obscura-blacklist/blacklist.conf`
- `/etc/obscura-blacklist/sources/`

Если нужно скопировать изменения source-файлов из клонированного репозитория в установленный blacklist и сразу применить их, используйте:

```bash
sudo sh scripts/refresh-blacklist.sh --repo
```

В режиме `--repo` скрипт копирует source-файлы из репозитория:
- `blacklist/config/sources/`

в:
- `/etc/obscura-blacklist/sources/`

и затем запускает обычное обновление через установленный конфиг `/etc/obscura-blacklist/blacklist.conf`.

## Как удалить модуль

Для удаления systemd-интеграции выполните:

```bash
sudo sh scripts/uninstall-blacklist.sh
```

Скрипт:
- отключит и остановит timer
- остановит service
- попытается убрать активные правила модуля
- удалит systemd units

Важно:
- скрипт удаляет именно systemd-интеграцию
- установленный конфиг и source-файлы не удаляются автоматически
- каталоги состояния и кэша тоже не удаляются автоматически

## Куда смотреть при проблемах

Сначала проверьте:
- запущен ли Docker
- выполняется ли команда `docker info`
- работает ли systemd на хосте

Если установка завершилась с ошибкой, исправьте указанную причину и запустите тот же скрипт ещё раз.
