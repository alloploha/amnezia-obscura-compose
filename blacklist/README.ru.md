# Чёрный список для Obscura и Amnezia VPN

**Amnezia VPN** и проект **Obscura** используются для развертывания различных VPN и прокси-сервисов на Linux-серверах.

Пользователи VPN применяют различные клиентские приложения для подключения к этим сервисам - в основном для обхода блокировок, накладываемых правительствами некоторых стран на доступ к иностранным сайтам, играм и файлам.

VPN-сервисы активно выявляются, а IP-адреса серверов вносятся в чёрные списки. Один из распространённых методов выявления VPN-серверов — использование приложений, установленных на устройствах пользователей (компьютерах, смартфонах, планшетах). Эти приложения обращаются к своим серверам через VPN и тем самым передают им IP-адрес VPN-сервера.

В совокупности с дополнительной информацией (например: часовой пояс устройства, язык системы, время суток и день недели) такие приложения и их серверы могут с высокой вероятностью определить:

* используется ли в данный момент VPN;
* какой IP-адрес принадлежит VPN-серверу.

Чёрные списки, размещённые на том же сервере, где запущены VPN-сервисы, блокируют любые исходящие соединения к серверам, которые могут способствовать выявлению VPN.

## Как это работает

**Amnezia VPN** и **Obscura** устанавливаются на Linux-сервер в виде Docker-контейнеров. Например, Amnezia VPN с протоколом XRay запускается в контейнере с именем `amnezia-xray`.

Docker использует `iptables` для маршрутизации и разделения сетевых потоков между контейнерами, хост-системой и интернетом. Дополнительные правила, например фильтрация по IP-адресам, задаются в цепочке `DOCKER-USER`.

Данный модуль содержит набор команд, предназначенных для заполнения цепочки `DOCKER-USER` IP-адресами, доступ к которым нежелателен.

Также модуль устанавливает фоновый процесс (демон), который обеспечивает постоянное наличие этих правил в таблице.

## Заметка про Obscura

**Obscura** - это проект, частью которого является модуль чёрных списков. При этом данный модуль не зависит от других файлов вне своего каталога.

В каталоге `scripts` находятся несколько bash-скриптов, упрощающих:

* установку,
* удаление,
* обновление чёрных списков.

Однако те же действия можно выполнить вручную, используя команды самого модуля.

## Установка

1. Скачайте проект на сервер:

```bash
git clone https://github.com/alloploha/amnezia-obscura-compose.git
```

2. Убедитесь, что установлены следующие компоненты (при необходимости установите их):

* `iptables`
* `ipset`
* `python3`

3. Выполните скрипт установки:

```bash
sudo scripts/install-blacklist.sh
```

Если какие-либо компоненты отсутствуют, скрипт уведомит об этом. Установите недостающие зависимости и запустите скрипт повторно.

4. Проверьте, что цепочка `DOCKER-USER` заполнена
   (при первом запуске это может занять 2–3 минуты):

```bash
sudo iptables -L DOCKER-USER -v
```

---

## Обновление

1. Обновите файлы проекта:

```bash
git pull
```

2. Выполните скрипт обновления:

```bash
sudo scripts/refresh-blacklist.sh
```

---

## Удаление

1. Выполните скрипт удаления:

```bash
sudo scripts/uninstall-blacklist.sh
```
